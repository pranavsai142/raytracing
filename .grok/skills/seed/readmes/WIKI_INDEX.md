# Project Wiki — Living Documentation

This is the home for the project's important mental models, invariants, and "how things actually work" explanations that are not obvious from the code.

## Principles
- Keep chapters short and high-signal.
- Focus on things that repeatedly cause confusion or bugs.
- Update whenever you have a clear "I wish I had known this earlier" insight.

## Suggested Structure
- Core concepts (the primitives)
- Key rules & invariants
- Common failure modes & how to avoid them
- Architecture decisions with rationale

Start simple. Add chapters as real understanding solidifies.
